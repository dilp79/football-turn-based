# Пошаговый футбол

Браузерный hot-seat прототип пошагового футбола для двух игроков на одном компьютере. В проекте реализованы:

- драфт состава на 100 монет из 20 карточек;
- расстановка пятёрок на поле 15x9;
- матчевый цикл на двух кубиках;
- перемещение, пас, удар, отбор, подкат и дриблинг;
- суперспособности сильных игроков;
- режим `vs ИИ`, где Лазурные Волки автоматически проводят драфт, расстановку и матчевые ходы;
- матчевые метрики: владение, удары, точность паса, дриблинг, отборы, сейвы, фолы и MVP;
- штрафные после фолов в подкате вместо простого досрочного завершения хода;
- лёгкие DOM-анимации для фаз матча, перемещений, пасов, ударов, стыков и голов;
- процедурный звук через Web Audio API с mute-toggle в верхней панели;
- пошаговый onboarding на первый запуск с принудительным `vs ИИ`, контекстными подсказками по драфту, расстановке, кубикам, действиям и финальной статистике;
- pixel-art визуал без внешних ассетов.

## Запуск

1. Поднять статический сервер:

```bash
npm run serve
```

2. Открыть в браузере `http://localhost:4173`.

3. В верхней панели выбрать режим:

- `2 игрока` — классический hot-seat;
- `vs ИИ` — человек играет за Алые Кометы, ИИ управляет Лазурными Волками.
- `Звук: вкл/выкл` — мгновенное отключение процедурных эффектов.

## Smoke test

```bash
npm test
```

Тест проверяет:

- легальный драфт;
- переход к расстановке;
- старт матча;
- подбор мяча с центра;
- успешный пас после первого перемещения.

Визуальные и звуковые эффекты smoke-тестом не ассертятся, но прогоняются через браузерные runtime-проверки и `ui:snapshots`.

## AI smoke test

```bash
npm run test:ai
```

Тест проверяет режим `vs ИИ` по цепочке:

- легальный драфт человека и ответные пики ИИ;
- автоматическую расстановку AI-команды;
- корректный переход в матч и исполнение первого AI-хода.

## Match systems test

```bash
npm run test:systems
```

Тест проверяет новые матчевые системы:

- запуск штрафного после фола в подкате;
- передачу отдельного хода пострадавшей стороне;
- очистку состояния штрафного после завершения розыгрыша;
- базовую матчевую статистику и связанный state flow.

## Tutorial smoke test

```bash
npm run test:tutorial
```

Тест проверяет onboarding как пользовательский flow:

- первый запуск сразу переводится в `vs ИИ`;
- human draft и ответный AI-пик разделяются tutorial overlay;
- AI не ходит, пока подсказка не закрыта;
- перезапуск обучения сбрасывает матч;
- `skip` ставит `tutorialComplete` в `localStorage`.

## Headless Browser

Headless Chromium для layout-проверок установлен через Playwright.

```bash
npm run ui:snapshots
```

Команда:

- поднимает локальную статическую страницу;
- проходит `draft -> placement -> match`;
- сохраняет скриншоты в `artifacts/layouts/`.

## Принятые игровые трактовки

В PRD были неоднозначные места, поэтому в прототипе зафиксированы такие правила:

- Два кубика дают `ОД = сумма`, а старший кубик становится `рывком`: это единоразовый бонус к дуэли или удару в текущем ходу.
- Вратари закреплены в створе: они не двигаются по полю, но после сейва начинают следующую атаку пасом.
- Подкат остаётся рискованным действием на чистом d6, но супер защитника расширяет успешный диапазон.
- После гола игроки возвращаются в стартовую расстановку, а разводит команда, которая пропустила.
- Матч идёт до 3 голов.

## Структура

- `index.html` — точка входа.
- `styles.css` — визуальная система и responsive-верстка.
- `src/game.js` — state machine и игровые правила.
- `src/ui.js` — рендер игрового интерфейса.
- `src/sprites.js` — генерация процедурных pixel-art спрайтов.
- `src/data.js` — каталог футболистов, команды и способности.
- `src/tutorial.js` — first-run onboarding и шаги обучения.
- `scripts/smoke-test.mjs` — Node smoke test без DOM.
- `docs/design-notes.md` — заметки по балансу, UX и графической модели.
